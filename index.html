Course: Git & Version Control
Project Title: Version Control Management of a Student Portal Using Git

Project Scenario

You are working as a software developer in a company developing a Student Portal Management System. Multiple developers work on different features simultaneously. Use Git to manage the project, collaborate with team members, track history, and maintain different versions of the software.

Project Objectives
At the end of this project, students should be able to:

Initialize a Git repository.
Track files using Git.
Create and manage branches.
Use Git Stash.
Work with remote repositories.
Merge branches. Create Git tags.
Cherry-pick commits.
Analyze commit history.
Undo commits safely.

Expected Project Outcome

After completing this project, students will be able to:

  1. Create and manage Git repositories.
  2. Track project files efficiently.
  3. Work with branches for feature development.
  4. Save temporary work using Git Stash.
  5. Collaborate using remote repositories.
  6. Merge branches and resolve conflicts.
  7. Tag project versions for releases.
  8. Transfer specific commits using cherry-pick.
  9. Analyze commit history using Git log and Git show.
  10. Filter commits by author and date.
  11. Undo changes safely using Git revert.

This integrated mini project combines all 12 Git exercises into a single, realistic workflow that simulates version control practices in a software development environment.
